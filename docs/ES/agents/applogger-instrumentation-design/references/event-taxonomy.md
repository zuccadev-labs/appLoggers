# Event Taxonomy

Cover at minimum:

1. Startup lifecycle.
2. Authentication outcomes.
3. Network anomalies.
4. Critical business failures.
5. Core performance metrics.

Use levels consistently:

1. `info` for expected milestones.
2. `warn` for recoverable anomalies.
3. `error` for user-visible failures.
4. `critical` for app-blocking failures.
5. `metric` for quantitative performance.
